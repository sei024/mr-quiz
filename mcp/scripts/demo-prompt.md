# デモ動画用プロンプト

## 撮影者向けメモ

- 動画は 2〜3分、非エンジニア含むターゲット
- ブログ記事の「こんなこともできます」セクション用
- プロンプトは2つに分かれている（事前確認 → 本番分析）

## 事前準備（撮影前に済ませておく）

```bash
cd mr-quiz-mcp
GOOGLE_CLOUD_PROJECT=your-gcp-project-id npx tsx scripts/seed-test-data.ts
```

---

## プロンプト1: 事前確認（そのまま Claude Code に入力）

```
mr-quiz の MCP ツールを使ったデモをこれから行います。
まず事前確認として、テストユーザー test-user-mcp のデータが正しく入っているか確認してください。

確認項目:
- get_user_profile でプロファイルが取得できるか
- get_user_stats で統計が取得できるか（totalQuizzes=15 になっているはず）

問題なければ「準備OKです。次のプロンプトで分析を開始してください」と教えてください。
```

### 期待される動き（〜30秒）

- Claude が 2 ツールを呼び出してデータ存在を確認
- 「準備OKです」と返答

---

## プロンプト2: 本番分析（確認OKの後に入力）

```
では、test-user-mcp さんの学習状況を分析して、今後の学習アドバイスをください。

具体的には:
1. まずプロファイルを見て、この人がどんなエンジニアか教えて
2. 次に全体の成績と、カテゴリごとの強み・弱みを分析して
3. 成長の記録も確認して、モチベーションにつながるポイントを見つけて
4. 最後に、この人のキャリア目標に合わせた具体的な学習アドバイスをまとめて
```

---

## 期待される流れ（2〜3分）

### Act 1: 「この人は誰？」（〜30秒）

Claude が `get_user_profile` を呼び出し、プロファイルを紹介:

> 経験3年のミドルエンジニアで、「セキュリティに強いバックエンドエンジニアになりたい」
> という目標を持っています。security と performance に注力したいとのこと。

**見どころ**: キャリア目標が日本語で表示され、人物像が伝わる

### Act 2: 「成績はどう？」（〜1分）

Claude が `get_user_stats` → `analyze_weak_categories` を呼び出し:

> 全体の正答率は67%。refactoring と logic は100%と完璧ですが、
> **security と performance が33%で弱点** です。
> しかも performance は下降傾向、security は改善傾向という違いがあります。

**見どころ**: 数字とトレンドで「AIがちゃんと分析してる感」が出る

### Act 3: 「成長してる？」（〜30秒）

Claude が `get_growth_milestones` を呼び出し:

> 3つのマイルストーンを達成しています:
> - 初めての正解
> - 累計10問回答
> - refactoring マスター（正答率100%）

**見どころ**: 成長記録が自動で追跡されている

### Act 4: 「で、どうすればいい？」（〜1分）

Claude が全情報を統合して学習アドバイスを生成:

> キャリア目標「セキュリティに強いバックエンドエンジニア」に対して、
> security は改善傾向なのでこの調子で。
> performance は下降傾向なので、N+1クエリなど基礎から復習を。
> 得意な refactoring の知識を security に応用する視点も有効です。

**見どころ**: プロファイル × 統計 × トレンド を掛け合わせたパーソナライズされた提案

---

## 演出のポイント

- **ツール呼び出しが自動で走る様子** が画面に映るのがMCPデモの醍醐味
- Claude の応答は毎回少し異なるが、上記の流れは安定して再現される
- プロンプト2 で 4つのツールが連鎖する「魔法感」を見せる
- 必要に応じて動画は倍速編集OK（ツール呼び出し待ち部分など）

```bash
GOOGLE_CLOUD_PROJECT=your-gcp-project-id npx tsx scripts/seed-test-data.ts --cleanup
```
